SHARING AND UPDATING


I started with github yesterday.I have cloned a repository which is a collabrating with 4 more people.
I faced a conflicting scenario while performing basic commands.And found the solution for the same.

SHARING AND UPDATING
When i was working on the same file as that one of the other collaborator.
I have made changes to that file and committed by changes without pushing the file.
In the mean time the other collaborator had already pushed that file and  when i was trying to push the file later, the terminal did'nt allow me to do so.

Now in order to resolve that conflict, I used 'git fetch origin' command,
that synchronized me  with a remote repository, fetching all the data it has that I do not,
followed by merging the contents by using 'git merge origin/master' and finally pushing it.

So from this i have understood that while collabrating with others developers in Git,first
 have to put all that data on a server that the other developers have access to.
Thus minimizing the occurences of conflicts.
